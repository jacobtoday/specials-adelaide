Phase 1

Your project files

Phase 2

Set up GitHub

Railway deploys from GitHub. You need the folder in a repo before anything goes live.

1
Create a GitHub account (if you don't have one)
Free — takes 2 minutes

Go to github.com and sign up. Use your Born email address.

2
Create a new repository
Where your code will live
  1. Click the + in the top-right → New repository
  2. Name it specials-adelaide
  3. Set it to Private (recommended — keeps your admin URL out of search engines)
  4. Leave everything else unchecked, click Create repository
3
Install Git and push your files
Run these commands in Terminal inside the project folder

Open Terminal (Mac: press ⌘Space, type Terminal). Navigate into your project folder:

cd ~/Downloads/specials-adelaide

Then run these commands one at a time, pressing Enter after each:

git init
git add .
git commit -m "Initial commit — Specials Adelaide"
git branch -M main
git remote add origin https://github.com/YOURUSERNAME/specials-adelaide.git
git push -u origin main
⚠️ Replace YOURUSERNAME with your actual GitHub username. GitHub will ask you to log in the first time — use your GitHub email and password.

When it finishes, refresh your GitHub repo page — you should see all 6 files listed.

Phase 3

Deploy on Railway

Railway will detect your package.json, install Express, and start the server automatically.

1
Create a Railway account
railway.com — free to start

Go to railway.com and click Login. Choose Login with GitHub — this links Railway to your GitHub so it can read your repos.

ℹ️ Railway gives you a free trial with $5 credit. After that, the Hobby plan is $5/month which comfortably covers a small site like this.
2
Create a new project
Takes about 3 minutes
  1. From the Railway dashboard, click New Project
  2. Choose Deploy from GitHub repo
  3. Select specials-adelaide from the list
  4. Click Deploy Now

Railway will automatically detect package.json, install Express, and run node server.js. The first deploy takes about 60–90 seconds.

✅ Watch the build logs in real time. You'll see "Specials Adelaide running on port 3000" when it's live.
3
Get your live URL
Railway gives you a free .up.railway.app URL immediately
  1. Click on your service in the Railway dashboard
  2. Go to Settings → Networking
  3. Click Generate Domain
  4. You'll get a URL like specials-adelaide.up.railway.app

Visit the URL — your site is live. The admin panel is at yourdomain.up.railway.app/admin.

4
Verify everything works
Quick checklist before moving on
  • ✅ Main site loads at / — specials, filters, hero all working
  • ✅ Admin panel loads at /admin — login screen appears
  • ✅ Admin login works with admin / specials2025
  • ✅ You can edit a special price and see it update
  • ✅ Week planner functions — can add specials, share plan
Phase 4

Custom domain (optional)

Replace the .up.railway.app URL with your own domain like specialsadelaide.com.au.

1
Buy a domain
~$20/yr for a .com.au

Good registrars for Australian domains:

  • Crazy Domains (crazydomains.com.au) — good for .com.au
  • Namecheap (namecheap.com) — good for .com
  • Google Domains → now Squarespace Domains

Suggested domain: specialsadelaide.com.au or adlspecials.com.au

2
Add domain to Railway
Railway handles SSL automatically
  1. In Railway: go to your service → Settings → Networking
  2. Click + Custom Domain
  3. Enter your domain, e.g. specialsadelaide.com.au
  4. Railway shows you a CNAME record to add — copy it

Then go to your domain registrar's DNS settings and add:

Type:  CNAME
Name:  @  (or www)
Value: [the value Railway gives you]
TTL:   3600

DNS changes take 5–30 minutes to propagate. Railway auto-provisions an SSL certificate (HTTPS) once it detects the DNS.

✅ Railway handles HTTPS automatically — you don't need to do anything for the SSL certificate.
Phase 5

Securing the admin panel

The admin panel at /admin has its own password — change it before going live.

1
Change the admin password
Do this before sharing the URL publicly
  1. Open /admin and log in with admin / specials2025
  2. Go to Settings in the left sidebar
  3. Change the Username to something other than "admin"
  4. Set a strong Password (at least 12 characters, mix of letters/numbers)
  5. Click Save next to each field
⚠️ The admin password is stored in localStorage in the browser — it's per-device. If you log in from a new device, it falls back to the hardcoded default. For production security, update the default credentials directly in admin.html before deploying (search for CREDENTIALS near the top of the script block).
2
Change the hardcoded default credentials (recommended)
One-line edit in admin.html before pushing

Open admin.html in a text editor. Find this line (near the bottom in the <script> block):

let CREDENTIALS={user:"admin",pass:"specials2025"};

Change it to your own username and a strong password:

let CREDENTIALS={user:"jacob",pass:"YourStrongPassword123!"};

Save the file, then push to GitHub (see Phase 6 below). Railway will redeploy automatically.

Phase 6

Pushing updates

Every time you push to GitHub, Railway redeploys automatically — usually in under 60 seconds.

1
Updating a file
The workflow for every future change

Make changes to index.html or admin.html, then run:

cd ~/Downloads/specials-adelaide
git add .
git commit -m "Updated specials data"
git push

Railway detects the push and redeploys. Watch the progress in the Railway dashboard — your live URL updates when the green checkmark appears.

💡 You can also use GitHub Desktop (desktop.github.com) if you prefer a visual interface over Terminal commands. It does the same thing with a point-and-click UI.
2
Adding new specials without coding
Use the admin panel

For day-to-day content changes — adding a new restaurant, updating a price, toggling a special off — use the admin panel at /admin. Changes save to the browser's localStorage on that device.

ℹ️ Important: Admin panel changes currently save to your browser only. To push changes to the live site for everyone, you'll need to connect Supabase (Phase 7 below) or manually update the HTML file. The admin panel's Export → Generate SQL button helps bridge this gap.
Phase 7 — Optional but recommended

Connect Supabase for live data

This makes admin edits update the live site for everyone, not just your browser.

1
Create your Supabase project
supabase.com — free tier
  1. Go to supabase.com → New project
  2. Name it specials-adelaide, pick ap-southeast-2 (Sydney)
  3. Save your database password somewhere safe
  4. Once created, go to Project Settings → API
  5. Copy your Project URL and anon public key
2
Run the database schema
Creates your tables in one go

In Supabase → SQL Editor → New query, paste and run this:

create extension if not exists "pgcrypto";

create table restaurants (
  id            uuid primary key default gen_random_uuid(),
  name          text not null,
  suburb        text not null,
  cuisine_type  text,
  address       text,
  phone         text,
  website       text,
  is_active     boolean not null default true,
  created_at    timestamptz not null default now()
);

create table specials (
  id              uuid primary key default gen_random_uuid(),
  restaurant_id   uuid not null references restaurants(id) on delete cascade,
  name            text not null,
  description     text,
  price           numeric(6,2) not null,
  category        text,
  session_type    text,
  days_available  text[] not null,
  available_from  text,
  available_until text,
  is_active       boolean not null default true,
  created_at      timestamptz not null default now()
);

create index on specials(restaurant_id);
create index on specials(is_active);

alter table restaurants enable row level security;
alter table specials enable row level security;

create policy "Public read restaurants" on restaurants
  for select using (is_active = true);
create policy "Public read specials" on specials
  for select using (is_active = true);
3
Push your data from the admin panel
Export → Generate SQL → run in Supabase
  1. In the admin panel, go to Import / Export
  2. Click Generate SQL
  3. Copy the generated SQL
  4. Paste it into Supabase SQL Editor and run it

This upserts all your restaurants and specials into the database.

4
Add Supabase credentials to Railway
Environment variables — never hardcode keys
  1. In Railway → your service → Variables tab
  2. Add these two environment variables:
SUPABASE_URL=https://yourproject.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5c...

Then in index.html, replace the static RESTAURANTS and SPECIALS arrays with Supabase fetch calls. We can help you do that in the next development session.

💡 The site works fully without Supabase right now — all 95 specials are baked in. Supabase is the upgrade that lets you manage data without touching code.
